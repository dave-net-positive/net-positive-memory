# Read

`vault_read` = body of one note. Most expensive rung (bodies enter context). Last, not first.

## When
- Search confirmed a specific note relevant.
- Need full body, not just the `summary` search already gave.

## How
- Read by path, e.g. `entities/bifrost.md`.
- `format: metadata` when only frontmatter/tags/stat needed. Cheaper than full body.
- 1 to 3 notes, not a folder. Want several related? Graph or structured search first to confirm which earn a read.

## Don't
- No reading `INDEX.md` + every linked note on a new session. INDEX + newest session, then stop.
- No re-reading a note already read this session unless it changed.
